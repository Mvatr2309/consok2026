# Спецификация: Сервис записи на групповые консультации МФТИ

## 1. Общее описание

Веб-сервис для записи абитуриентов на групповые онлайн-консультации с экспертами по образовательным программам МФТИ (онлайн-магистратура «Разработка IT-продукта»).

### Основные сценарии использования

1. **Абитуриент** получает прямую ссылку на конкретную программу, выбирает эксперта и слот, заполняет форму — записывается на консультацию.
2. **Администратор** управляет программами, экспертами, слотами и записями через защищённую админ-панель.
3. **Система** автоматически отправляет уведомления (email + Telegram): подтверждение записи, напоминание перед встречей, уведомление об отмене.

---

## 2. Роли пользователей

| Роль | Доступ | Аутентификация |
|------|--------|----------------|
| Абитуриент | `/programs/[id]`, `/api/bookings` | Без авторизации |
| Администратор | `/admin`, `/api/admin/*` | HTTP Basic Auth |

---

## 3. Функциональные требования

### 3.1 Публичная часть (абитуриент)

#### Страница программы (`/programs/[id]`)

- Заголовок: «Запись на групповую консультацию»
- Название и описание программы
- Список экспертов с фото и описанием (карточки)
- При выборе эксперта — список всех доступных будущих слотов (дата, время МСК, свободные места)
- Форма записи: ФИО, Email
- Валидация: корректный email, 1 запись на программу с одного email
- Согласие на обработку персональных данных, запись встречи, cookie
- Экран успеха: данные консультации, ссылка на встречу, кнопки «Google Календарь» / «Скачать .ics» / «Напоминание в Telegram»

#### Главная страница (`/`)

- Каталог всех программ с карточками (название, описание, эксперты, ближайшие слоты)
- Ссылки на страницы отдельных программ
- Вторичный сценарий — основной способ распространения: прямые ссылки `/programs/[id]`

### 3.2 Админ-панель (`/admin`)

Защищена HTTP Basic Auth. Содержит 6 разделов:

#### Программы
- CRUD: создание, редактирование, удаление программ
- Привязка экспертов к программам (many-to-many)
- Отображение прямой ссылки для абитуриентов с кнопкой «Копировать»

#### Эксперты
- CRUD: ФИО, описание, фото (загрузка файла), ссылка на встречу
- Привязка к программам (many-to-many)

#### Слоты
- CRUD: дата/время, макс. участников, программа, эксперт
- Два вида отображения: таблица и календарь
- Пагинация: 10 слотов на страницу (табличный вид)
- Массовый импорт через CSV (разделитель `;`)
- Скачивание шаблона CSV
- Массовое удаление с чекбоксами

#### Записи
- Просмотр всех записей (имя, email, программа, эксперт, дата, время записи)
- Пагинация: 10 записей на страницу
- Удаление записей (с отправкой уведомления об отмене на email и в Telegram)

#### Согласия
- Просмотр логов согласий пользователей (персональные данные, запись встречи, cookie)

#### Уведомления
- Включение/выключение email при регистрации
- Включение/выключение напоминаний
- Настройка времени отправки напоминания (в минутах до встречи)

### 3.3 Уведомления (Email + Telegram)

#### Подтверждение записи (email)
- Триггер: сразу после создания записи (асинхронно)
- Содержимое: программа, эксперт, дата/время, кнопки «Google Календарь» / «Скачать .ics»
- Управление: вкл/выкл в настройках админки

#### Напоминание перед встречей (email + Telegram)
- Триггер: cron-скрипт (`scripts/send-reminders.ts`)
- Содержимое: программа, эксперт, дата/время, кнопка «Перейти на встречу»
- Управление: вкл/выкл + время в минутах (настройки админки)
- Каждая запись получает напоминание только 1 раз (`reminderSentAt`)

#### Уведомление об отмене (email + Telegram)
- Триггер: удаление записи администратором
- Содержимое: программа, эксперт, дата/время, сообщение об отмене

### 3.4 Telegram-бот (@mipt_consultation_bot)

- Бот для напоминаний о консультациях
- Подписка через deep link с сайта (кнопка «Напоминание в Telegram»)
- Защита от спама: реагирует только на `/start` с deep link
- При подписке отправляет: детали, ссылку на встречу, кнопки Google Календарь и .ics
- Напоминание за 1 час до встречи со ссылкой на встречу
- Уведомление при отмене консультации

### 3.5 CSV-импорт слотов

Формат файла:
```
Эксперт;Программа;Дата и время (ГГГГ-ММ-ДД ЧЧ:ММ);Макс. участников
Иванов Иван Иванович;Цифровой продукт 1;2026-03-20 10:00;5
```

- Разделитель: `;` (или `,`)
- Маппинг эксперта/программы по имени (регистронезависимо) или по ID
- Строки с `#` игнорируются (комментарии)
- Возвращает количество созданных слотов и список ошибок

## 4. Модель данных

### Product (Программа)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | Название программы |
| description | String | Описание |
| experts | Expert[] (M2M) | Связанные эксперты |
| slots | Slot[] | Слоты этой программы |

### Expert (Эксперт)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | ФИО |
| photo | String | Путь к фото (`/experts/...`) |
| description | String | Описание |
| meetingLink | String | Ссылка на онлайн-встречу |
| accessToken | String, unique | Токен для кабинета эксперта |
| login | String?, unique | Логин для входа |
| passwordHash | String? | Хеш пароля |
| products | Product[] (M2M) | Связанные программы |
| slots | Slot[] | Слоты этого эксперта |

### Slot (Слот)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| dateTime | DateTime | Дата и время (UTC) |
| maxParticipants | Int | Макс. кол-во участников |
| productId | Int, FK | Программа |
| expertId | Int, FK | Эксперт |
| bookings | Booking[] | Записи на этот слот |

### Booking (Запись)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | ФИО абитуриента |
| email | String | Email абитуриента |
| slotId | Int, FK | Слот |
| createdAt | DateTime | Дата создания |
| reminderSentAt | DateTime? | Когда отправлено напоминание |
| attended | Boolean (default: false) | Отметка о присутствии |

### Settings (Настройки)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK | Идентификатор |
| key | String, unique | Ключ настройки |
| value | String | Значение |

Используемые ключи настроек:
- `registration_email_enabled` — `"true"` / `"false"`
- `reminder_enabled` — `"true"` / `"false"` (по умолчанию включено: отсутствие настройки трактуется как `"true"`)
- `reminder_minutes_before` — число минут (по умолчанию `"60"`)

### TelegramSub (Подписка на Telegram-напоминание)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| chatId | String | Telegram chat ID |
| bookingId | Int, unique, FK | Связанный букинг |
| createdAt | DateTime | Дата подписки |

### Consent (Согласие)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| bookingId | Int, FK | Связанный букинг |
| personalData | Boolean | Согласие на обработку ПД |
| meetingRecording | Boolean | Согласие на запись встречи |
| cookiePolicy | Boolean | Согласие с cookie |
| ip | String | IP-адрес |
| userAgent | String | User-Agent |
| createdAt | DateTime | Дата |

### Связи
- Product ↔ Expert: many-to-many (`_ExpertToProduct`)
- Product → Slot: one-to-many
- Expert → Slot: one-to-many
- Slot → Booking: one-to-many
- Booking → TelegramSub: one-to-one (optional)
- Booking → Consent: one-to-many

## 5. API-эндпоинты

### Публичные

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/bookings` | Создание записи |
| GET | `/api/bookings/[id]/ics` | Скачать .ics файл |

**POST /api/bookings**
- Body: `{ slotId, name, email, programId, programName, consents }`
- Валидации: обязательные поля, формат email, 1 email на программу, наличие свободных мест
- Ответ: `{ meetingLink, expertName, dateTime, programName, bookingId }`
- Ошибки: 400, 404, 409, 500

**GET /api/bookings/[id]/ics**
- Возвращает .ics файл (iCalendar) для добавления в любой календарь
- Content-Type: `text/calendar; charset=utf-8`

### Кабинет эксперта

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/expert/login` | Вход по логину/паролю |
| POST | `/api/expert/logout` | Выход |
| GET | `/api/expert` | Данные эксперта (по cookie или token) |
| PATCH | `/api/expert/attendance` | Отметка присутствия абитуриента |

**PATCH /api/expert/attendance**
- Body: `{ bookingId, attended }` (attended — boolean)
- Проверяет, что запись принадлежит эксперту
- Обновляет поле `attended` в записи
- Ошибки: 400, 403, 404

## 6. Бизнес-правила

1. Один email может быть записан на одну программу только один раз
2. Запись невозможна, если все места в слоте заняты
3. На странице программы и главной странице отображаются все будущие слоты (`dateTime >= now`) без ограничения по количеству дней
4. Ссылка на встречу (`meetingLink`) привязана к эксперту, а не к слоту
5. Уведомления отправляются асинхронно (fire-and-forget), ошибка отправки не блокирует запись
6. Напоминание отправляется однократно — отслеживается через `reminderSentAt`
7. При удалении записи каскадно удаляются согласия и Telegram-подписки
8. Telegram-бот реагирует только на deep link — защита от спама
