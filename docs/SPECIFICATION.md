# Спецификация: Сервис записи на групповые консультации МФТИ

## 1. Общее описание

Веб-сервис для записи абитуриентов на групповые онлайн-консультации с экспертами по образовательным программам МФТИ (онлайн-магистратура «Разработка IT-продукта»).

### Основные сценарии использования

1. **Абитуриент** получает прямую ссылку на конкретную программу, выбирает эксперта и слот, заполняет форму — записывается на консультацию.
2. **Администратор** управляет программами, экспертами, слотами и записями через защищённую админ-панель.
3. **Система** автоматически отправляет email-уведомления: подтверждение записи и напоминание перед встречей.

---

## 2. Роли пользователей

| Роль | Доступ | Аутентификация |
|------|--------|----------------|
| Абитуриент | `/programs/[id]`, `/api/bookings` | Без авторизации |
| Администратор | `/admin`, `/api/admin/*` | HTTP Basic Auth |

---

## 3. Функциональные требования

### 3.1 Публичная часть (абитуриент)

#### Страница программы (`/programs/[id]`)

- Заголовок: «Запись на групповую консультацию»
- Название и описание программы
- Список экспертов с фото и описанием (карточки)
- При выборе эксперта — список доступных слотов (дата, время МСК, свободные места)
- Форма записи: ФИО, Email
- Валидация: корректный email, 1 запись на программу с одного email
- Экран успеха: данные консультации, ссылка на встречу, кнопки «Google Календарь» / «Яндекс Календарь»

#### Главная страница (`/`)

- Каталог всех программ с карточками (название, описание, эксперты, ближайшие 5 слотов)
- Ссылки на страницы отдельных программ
- Вторичный сценарий — основной способ распространения: прямые ссылки `/programs/[id]`

### 3.2 Админ-панель (`/admin`)

Защищена HTTP Basic Auth. Содержит 5 разделов:

#### Программы
- CRUD: создание, редактирование, удаление программ
- Привязка экспертов к программам (many-to-many)
- Отображение прямой ссылки для абитуриентов с кнопкой «Копировать»

#### Эксперты
- CRUD: ФИО, описание, фото (загрузка файла), ссылка на встречу
- Привязка к программам (many-to-many)

#### Слоты
- CRUD: дата/время, макс. участников, программа, эксперт
- Массовый импорт через CSV (разделитель `;`)
- Скачивание шаблона CSV

#### Записи
- Просмотр всех записей (имя, email, программа, эксперт, дата, время записи)
- Удаление записей

#### Уведомления
- Включение/выключение email при регистрации
- Включение/выключение напоминаний
- Настройка времени отправки напоминания (в минутах до встречи)

### 3.3 Email-уведомления

#### Подтверждение записи
- Триггер: сразу после создания записи (асинхронно)
- Содержимое: программа, эксперт, дата/время, кнопки «Google Календарь» / «Яндекс Календарь»
- Управление: вкл/выкл в настройках админки

#### Напоминание перед встречей
- Триггер: cron-скрипт (`scripts/send-reminders.ts`)
- Содержимое: программа, эксперт, дата/время, кнопка «Перейти на встречу»
- Управление: вкл/выкл + время в минутах (настройки админки)
- Каждая запись получает напоминание только 1 раз (`reminderSentAt`)

### 3.4 CSV-импорт слотов

Формат файла:
```
Эксперт;Программа;Дата и время (ГГГГ-ММ-ДД ЧЧ:ММ);Макс. участников
Иванов Иван Иванович;Цифровой продукт 1;2026-03-20 10:00;5
```

- Разделитель: `;` (или `,`)
- Маппинг эксперта/программы по имени (регистронезависимо) или по ID
- Строки с `#` игнорируются (комментарии)
- Возвращает количество созданных слотов и список ошибок

## 4. Модель данных

### Product (Программа)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | Название программы |
| description | String | Описание |
| experts | Expert[] (M2M) | Связанные эксперты |
| slots | Slot[] | Слоты этой программы |

### Expert (Эксперт)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | ФИО |
| photo | String | Путь к фото (`/experts/...`) |
| description | String | Описание |
| meetingLink | String | Ссылка на онлайн-встречу |
| products | Product[] (M2M) | Связанные программы |
| slots | Slot[] | Слоты этого эксперта |

### Slot (Слот)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| dateTime | DateTime | Дата и время (UTC) |
| maxParticipants | Int | Макс. кол-во участников |
| productId | Int, FK | Программа |
| expertId | Int, FK | Эксперт |
| bookings | Booking[] | Записи на этот слот |

### Booking (Запись)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK, autoincrement | Идентификатор |
| name | String | ФИО абитуриента |
| email | String | Email абитуриента |
| slotId | Int, FK | Слот |
| createdAt | DateTime | Дата создания |
| reminderSentAt | DateTime? | Когда отправлено напоминание |

### Settings (Настройки)
| Поле | Тип | Описание |
|------|-----|----------|
| id | Int, PK | Идентификатор |
| key | String, unique | Ключ настройки |
| value | String | Значение |

Используемые ключи настроек:
- `registration_email_enabled` — `"true"` / `"false"`
- `reminder_enabled` — `"true"` / `"false"`
- `reminder_minutes_before` — число минут (по умолчанию `"60"`)

### Связи
- Product ↔ Expert: many-to-many (`_ExpertToProduct`)
- Product → Slot: one-to-many
- Expert → Slot: one-to-many
- Slot → Booking: one-to-many

## 5. API-эндпоинты

### Публичные

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/bookings` | Создание записи |

**POST /api/bookings**
- Body: `{ slotId, name, email, programId, programName }`
- Валидации: обязательные поля, формат email, 1 email на программу, наличие свободных мест
- Ответ: `{ meetingLink, expertName, dateTime, programName }`
- Ошибки: 400, 404, 409, 500

## 6. Бизнес-правила

1. Один email может быть записан на одну программу только один раз
2. Запись невозможна, если все места в слоте заняты
3. На странице программы отображаются только будущие слоты (`dateTime >= now`)
4. Ссылка на встречу (`meetingLink`) привязана к эксперту, а не к слоту
5. Email-уведомления отправляются асинхронно (fire-and-forget), ошибка отправки не блокирует запись
6. Напоминание отправляется однократно — отслеживается через `reminderSentAt`
